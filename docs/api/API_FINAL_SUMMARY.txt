Q: How do I authenticate requests?
A: Include JWT token in header: Authorization: Bearer <token>
   Get token from: POST /api/auth/login

Q: Are all endpoints protected?
A: Yes, all 12 endpoints require valid JWT token

Q: Can I use SQL injection?
A: No, all queries are parameterized using $1, $2, etc.

Q: How do I handle errors?
A: Check response.success flag
   If false, display response.message to user
   Handle specific HTTP status codes (400, 401, 404, etc.)

Q: What's the base URL for the API?
A: http://localhost:3000/api (development)
   https://yourdomain.com/api (production)

Q: Can I filter results?
A: Yes, many endpoints support query parameters
   Examples: ?month=2025-01, ?status=pending, ?limit=10

Q: What if I need to add a new endpoint?
A: 1. Create method in appropriate controller
   2. Add route in routes/api.js with authenticateToken
   3. Follow same patterns (error handling, validation)
   4. Update documentation
